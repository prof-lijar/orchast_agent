#### **1. Linear Path (ပုံမှန် အစဉ်လိုက် လမ်းကြောင်း)**

*   **ရည်ရွယ်သူ:** Generative AI နှင့် Prompt Engineering ကို လုံးဝအခြေခံမှ စတင်လိုသူများ၊ နည်းပညာအတွေ့အကြုံ နည်းပါးသော လုပ်ငန်းရှင်များနှင့် ကျောင်းသားများ။
*   **လမ်းညွှန်:** Module 1 မှ စတင်၍ Module 4 အထိ တစ်ခုပြီးတစ်ခု အစဉ်အတိုင်း လေ့လာရန်။ Module တစ်ခုချင်းစီ၏ ပို့ချချက်များ၊ စမ်းသပ်မေးခွန်းများနှင့် လက်တွေ့လေ့ကျင့်ခန်းများကို အပြည့်အဝ ပြီးဆုံးအောင် ဆောင်ရွက်ပါ။
*   **အကြံပြုချက်:** Module 2 ပါ Prompting နည်းစနစ်များ (Zero-shot, Few-shot) ကို လက်တွေ့ AI Tools များတွင် အကြိမ်များစွာ လေ့ကျင့်ပါ။

#### **2. Accelerated Path (အရှိန်မြှင့် လမ်းကြောင်း)**

*   **ရည်ရွယ်သူ:** AI Tools (ဥပမာ- ChatGPT) ကို အခြေခံအားဖြင့် အသုံးပြုဖူးပြီးဖြစ်သော Freelancer များ သို့မဟုတ် နည်းပညာအခြေခံ အသင့်အတင့်ရှိသူများ။
*   **ရှောင်ရှားနိုင်သော Module များ (Conditions):**
    *   **Module 1:**
        *   **Condition:** Generative AI tools (ChatGPT, Gemini) ကို အသုံးပြုဖူးပြီး အခြေခံ သဘောတရားများကို နားလည်ပြီးပါက **1.1 နှင့် 1.2** ကို အမြန်ဖတ်ရှုနိုင်သည် (Skim)။
        *   **မဖြစ်မနေ လေ့လာရမည်:** **1.3 (LLM အလုပ်လုပ်ပုံ)** ကို သေချာစွာ လေ့လာပါ။
    *   **Module 2:**
        *   **Condition:** Prompt များ ရေးသားရာတွင် `Instruction` နှင့် `Context` ကို ခွဲခြားနားလည်ပြီးပါက **2.1 နှင့် 2.2** ကို အမြန်ဖတ်ရှုနိုင်သည်။
        *   **မဖြစ်မနေ လေ့လာရမည်:** **2.3 (Prompt Refinement)** ကို အသေးစိတ်လေ့လာပြီး လက်တွေ့လေ့ကျင့်ပါ။
*   **အကြံပြုချက်:** Module 3 (Advanced Prompting) နှင့် Module 4 (Business Integration) တို့တွင် အချိန်ပိုပေး၍ လက်တွေ့လုပ်ဆောင်မှုများကို အာရုံစိုက်လေ့လာပါ။

#### **3. Key Dependencies (အဓိက အမှီပြု Module များ)**

သင်တန်း၏ အယူအဆများသည် အောက်ပါအတိုင်း အပြန်အလှန် ဆက်နွှယ်နေပါသည်။

*   **Module 3** (Advanced Prompting Techniques) သည် **Module 2** (Prompt Engineering Basics) တွင် သင်ကြားခဲ့သော Prompt ၏ ဖွဲ့စည်းပုံကို အခြေခံထားသည်။ **Role-based** သို့မဟုတ် **CoT** ကို ထိရောက်စွာ အသုံးပြုနိုင်ရန် Module 2 ၏ Few-shot Prompting နည်းစနစ်ကို ကျွမ်းကျင်ရမည်။
*   **Module 4** (Business Integration & Ethics) သည် **Module 3** ၏ အဆင့်မြင့် Prompting နည်းလမ်းများကို လက်တွေ့လုပ်ငန်းခွင် စီမံခန့်ခွဲမှုပုံစံများတွင် အသုံးချခြင်းဖြစ်သည်။ ထို့ကြောင့် Module 3 ကို ကျော်သွားပါက စီးပွားရေးလုပ်ငန်း အလိုအလျောက်စနစ် (Automation) ပိုင်းတွင် ထိရောက်စွာ အသုံးချနိုင်မည် မဟုတ်ပါ။
*   သင်တန်းတစ်ခုလုံးသည် **Module 1** တွင် သင်ကြားသည့် AI ၏ ကန့်သတ်ချက်များကို နားလည်မှုအပေါ် အခြေခံထားသည်။ AI ၏ စွမ်းရည်နှင့် ကန့်သတ်ချက်များကို မသိပါက ကျင့်ဝတ်နှင့်ဆိုင်သော အန္တရာယ်များကို သတိမပြုမိဘဲ ဖြစ်လာနိုင်သည်။

| **AI (Artificial Intelligence):** လူသားတို့၏ ဉာဏ်ရည်ဉာဏ်သွေး လိုအပ်သည့် လုပ်ငန်းများကို လုပ်ဆောင်ရန် ကွန်ပျူတာစနစ်များကို ဖန်တီးခြင်း။ ၎င်းသည် ကျယ်ပြန့်သော နယ်ပယ်တစ်ခုဖြစ်သည်။ | စိုက်ပျိုးရေးလုပ်ငန်းတစ်ခုကို စီမံခန့်ခွဲသည့် **AI Factory** တစ်ခုဟု စဉ်းစားပါ။ AI သည် ပန်းတိုင် (အထွက်နှုန်းတိုးရေး) ကို သတ်မှတ်ပေးသည်။ | AI သည် အကြီးဆုံး နယ်ပယ်ဖြစ်ပြီး လူသား၏ စဉ်းစားပုံကို အတုယူရန် ကြိုးစားသည့် နည်းပညာအားလုံးကို ခြုံငုံမိသည်။ |
| **ML (Machine Learning):** ကွန်ပျူတာများကို ဒေတာများမှတစ်ဆင့် အစီအစဉ်များ (Program) ကို ရှင်းလင်းစွာ ရေးစရာမလိုဘဲ အလိုအလျောက် သင်ယူစေသော AI ၏ နယ်ပယ်ခွဲတစ်ခု။ | **ML Machines** များသည် ရာသီဥတုအချက်အလက် (ဒေတာ) ကို ကြည့်ရှုပြီး မည်သည့်အချိန်တွင် ရေလောင်းသင့်သည်ကို သင်ယူသည်။ | ML သည် ဒေတာကို အသုံးပြု၍ သင်ယူခြင်းနှင့် ခန့်မှန်းခြင်းအပေါ် အဓိကထားသည်။ |
| **DL (Deep Learning):** လူ့ဦးနှောက်၏ အာရုံကြောကွန်ရက်ပုံစံ (Neural Networks) ကို တုပဖန်တီးထားသော အလွှာများစွာရှိသည့် ML ၏ နယ်ပယ်ခွဲ။ ၎င်းသည် ရုပ်ပုံများနှင့် စာသားကဲ့သို့ ရှုပ်ထွေးသော ဒေတာများကို ပိုမိုကောင်းမွန်စွာ ကိုင်တွယ်နိုင်သည်။ | **DL Networks** များသည် မြေဆီလွှာဓာတ်ပုံများကို အလွန်တိကျစွာ ခွဲခြမ်းစိတ်ဖြာပြီး ပိုးမွှားရှိမရှိကို အလိုအလျောက် ဆုံးဖြတ်သည်။ Generative AI အများစုသည် DL ကို အသုံးပြုသည်။ | Generative AI သည် DL ပေါ်တွင် အခြေခံထားပြီး "အသစ်သော" (Original) အကြောင်းအရာများကို ဖန်တီးရန် အထူးပြုသည်။ |

#### **L 1.2. Generative AI Tools များ မိတ်ဆက် (ChatGPT, Gemini, Midjourney)**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Generative AI:** အချက်အလက်များ (Text, Images, Audio, Code) ကို "ဖန်တီး" သို့မဟုတ် "ထုတ်လုပ်" နိုင်သော AI အမျိုးအစား။ ၎င်းသည် ရှိပြီးသား ဒေတာများကို ခွဲခြမ်းစိတ်ဖြာကာ အသစ်အဆန်းများကို ပြန်လည်ပြုလုပ်သည်။ | Generative AI သည် လူတစ်ဦး၏ ပုံစံတူ ပန်းချီကားကို ရေးဆွဲပေးနိုင်သော ကျွမ်းကျင်ပန်းချီဆရာနှင့် တူသည်။ | Generative AI သည် ခွဲခြမ်းစိတ်ဖြာခြင်း (Analytical) သို့မဟုတ် ခန့်မှန်းခြင်း (Predictive) AI နှင့် ကွဲပြားပြီး အဓိကအားဖြင့် အကြောင်းအရာအသစ်များ ဖန်တီးပေးသည်။ |
| **Text-Based Models:** ChatGPT, Gemini, Claude ကဲ့သို့သော စာသားကို နားလည်၍ စာသားဖြင့် အဖြေပေးသည့် AI များ။ ၎င်းတို့သည် စာရေးခြင်း၊ အကျဉ်းချုပ်ခြင်း၊ ဘာသာပြန်ခြင်းတို့တွင် အဓိကအသုံးဝင်သည်။ | သင်သည် ChatGPT ကို သုံး၍ ဖောက်သည်များအတွက် မေးလေ့ရှိသောမေးခွန်းများ (FAQ) စာရင်းကို အမြန်ရေးခိုင်းနိုင်သည်။ ၎င်းသည် စီးပွားရေးလုပ်ငန်းများအတွက် အထူးအသုံးဝင်သည်။ | Text AI များသည် နေ့စဉ်လုပ်ငန်းခွင်သုံး စာရွက်စာတမ်းများနှင့် ဆက်သွယ်ရေးကိစ္စများအတွက် အားအကိုးရဆုံးဖြစ်သည်။ |
| **Image & Code Generation Models:** Midjourney, DALL-E (ပုံများဖန်တီးရန်) နှင့် GitHub Copilot (Code ဖန်တီးရန်) တို့ကဲ့သို့ AI များ။ ၎င်းတို့သည် သတ်မှတ်ထားသော ညွှန်ကြားချက်များမှ မူရင်းရုပ်ပုံများ သို့မဟုတ် software code များကို ထုတ်ပေးသည်။ | Midjourney ကို အသုံးပြု၍ သင့်ဝက်ဘ်ဆိုက်အတွက် "ခရမ်းရောင်အမိုးပါသော မြန်မာရိုးရာအိမ်" ပုံကို စက္ကန့်ပိုင်းအတွင်း ဖန်တီးနိုင်သည်။ | AI Tools မျိုးစုံသည် မတူညီသော Output များ (စာသား၊ ရုပ်ပုံ၊ code) အတွက် အထူးပြုထားသောကြောင့် မိမိလိုချင်သော ရလဒ်နှင့် ကိုက်ညီသော Tool ကို ရွေးချယ်ရမည်။ |

#### **L 1.3. LLM များ မည်သို့ အလုပ်လုပ်သနည်း (Large Language Models)**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Tokenization:** LLM များသည် စာသားများကို စကားလုံးအပြည့်အစုံမဟုတ်ဘဲ အသေးစား အစိတ်အပိုင်းများ (Tokens) အဖြစ် ခွဲခြမ်းစိတ်ဖြာသည်။ Tokens များသည် စကားလုံးများ၊ စာလုံးများ သို့မဟုတ် သတ်ပုံများ ဖြစ်နိုင်သည်။ | "မြန်မာပြည်" ဆိုသော စကားလုံးသည် "မြန်"၊ "မာ" နှင့် "ပြည်" ကဲ့သို့သော Tokens သုံးခု သို့မဟုတ် နှစ်ခု ဖြစ်နိုင်သည်။ AI သည် ၎င်း Tokens များကို အခြေခံ၍ သင်ယူသည်။ | AI သည် စကားလုံးများဖြင့် စဉ်းစားခြင်းမဟုတ်ဘဲ Tokens များဖြင့် တွက်ချက်သည်။ ၎င်း Tokens များ၏ ဆက်စပ်မှုကို သိရှိနားလည်ခြင်းဖြင့် အဓိပ္ပာယ်ကို နားလည်သည်။ |
| **Next Word Prediction:** LLM များသည် ၎င်းတို့၏ ကြီးမားသော လေ့ကျင့်ရေးဒေတာ (Training Data) မှတစ်ဆင့် ယခင် Tokens များကို အခြေခံ၍ နောက်ဆုံးပေါ်လာနိုင်ခြေရှိသော Token သို့မဟုတ် စကားလုံးကို ခန့်မှန်းခြင်းဖြင့် အလုပ်လုပ်သည်။ | သင်သည် AI ကို "ကျွန်တော့်ရဲ့ အကြိုက်ဆုံးအရောင်က..." ဟု ပြောပါက၊ AI သည် ဒေတာတွင် အများဆုံး တွေ့ရှိရသော ဆက်စပ်စကားလုံး (ဥပမာ: "အနီ" သို့မဟုတ် "အပြာ") ကို ခန့်မှန်း၍ ဆက်ရေးသည်။ | LLM များသည် ကြီးမားသော စာရင်းအင်းနှင့် ခန့်မှန်းခြင်း ယန္တရားများဖြစ်သည်။ ၎င်းတို့သည် "နားလည်ခြင်း" ထက် "အကောင်းဆုံး ဆက်စပ်မှု" ကို ရှာဖွေပေးခြင်း ဖြစ်သည်။ |
| **Context Window & 'Memory':** LLM များသည် စကားပြောဆိုမှုတစ်ခုလုံးကို အမြဲတမ်း မှတ်မိနိုင်ခြင်း မရှိပါ။ ၎င်းတို့သည် လက်ရှိ ဆွေးနွေးနေသည့် စကားပြောဆိုမှု၏ အပိုင်းအစ (Context Window) ကိုသာ မှတ်မိနိုင်သည်။ ၎င်းသည် ကန့်သတ်ချက်တစ်ခု ဖြစ်သည်။ | စားပွဲတစ်ခုပေါ်ရှိ မှတ်စုတိုလေးတစ်ခုနှင့် တူသည်။ ထိုမှတ်စုတိုလေးကိုသာ ဖတ်နိုင်သည်၊ စားပွဲပေါ်က စာအုပ်အားလုံးကို တစ်ချိန်တည်း ဖတ်မရ။ စကားပြောကြာလာသည်နှင့်အမျှ အစောပိုင်းပြောဆိုချက်များကို AI မေ့သွားနိုင်သည်။ | AI ၏ မှတ်ဉာဏ်သည် ကန့်သတ်ချက်ရှိသည်။ အရေးကြီးသော အချက်အလက်များကို Prompt အသစ်များတွင် ထပ်ခါတလဲလဲ ထည့်သွင်းပေးခြင်းသည် ရလဒ်ကောင်းစေသည်။ |

### Module 2: Prompt Engineering အခြေခံနှင့် နည်းစနစ်များ

#### **L 2.1. ထိရောက်သော Prompt တစ်ခု၏ ဖွဲ့စည်းပုံ: အခြေခံ လေးချက်**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **1. Instruction (ညွှန်ကြားချက်):** AI ကို မည်သည့်အလုပ် လုပ်စေလိုကြောင်း ရှင်းလင်းပြတ်သားစွာ ပြောပြခြင်း။ ၎င်းသည် Prompt ၏ အဓိက အစိတ်အပိုင်းဖြစ်သည်။ | **ညွှန်ကြားချက်:** "ကျွန်တော့်ရဲ့ မနက်ဖြန် အစည်းအဝေးအတွက် အရေးကြီးတဲ့ အချက် ၃ ချက်ကို အကျဉ်းချုပ်ပေးပါ"။ | ညွှန်ကြားချက်သည် ပထမဆုံးနှင့် အရှင်းလင်းဆုံး ဖြစ်ရမည်။ AI ကို 'ဘာလုပ်ရမယ်' ဆိုတာ တိတိကျကျ သိအောင် ပြောပြပါ။ |
| **2. Context (နောက်ခံအခြေအနေ):** AI အား ထိုအလုပ်ကို လုပ်ဆောင်ရန် လိုအပ်သည့် နောက်ခံအချက်အလက်များ သို့မဟုတ် အဘယ်ကြောင့် လုပ်ဆောင်သည်ကို ပြောပြခြင်း။ ၎င်းသည် အရည်အသွေးမြင့်မားသော အဖြေအတွက် မရှိမဖြစ် လိုအပ်သည်။ | **နောက်ခံအခြေအနေ:** "ဒီအစည်းအဝေးက စျေးကွက်ရှာဖွေရေး အဖွဲ့နဲ့ဖြစ်ပြီး၊ ကျွန်တော်က CEO ကို တင်ပြရမှာမို့ အသုံးအနှုန်းကို ပရော်ဖက်ရှင်နယ် ဆန်စေချင်တယ်"။ | AI ကို အခြေအနေပေးခြင်းဖြင့် ရရှိလာမည့် အဖြေသည် သင့်လုပ်ငန်း သို့မဟုတ် အခြေအနေနှင့် ပိုမို ကိုက်ညီလာမည်။ |
| **3. Example (နမူနာ):** သင်လိုချင်သည့် ရလဒ်ပုံစံကို AI အား ပြသခြင်း (Few-shot Prompting ၏ အခြေခံ)။ | **နမူနာ:** "ဒီလိုမျိုး စာသားပုံစံအတိုင်း ရေးပေးပါ - [နမူနာ စာသား/ဖော်မတ်ကို ဤနေရာတွင် ထည့်သွင်းပါ]"။ | AI ကို အကျိုးရှိရှိ လမ်းညွှန်ရန် နမူနာပုံစံကို ပြသခြင်းသည် စာသား၊ စာရင်း သို့မဟုတ် code ဖော်မတ်များအတွက် အထူးထိရောက်သည်။ |
| **4. Format (ဖော်မတ်):** AI အား အဖြေကို မည်သည့်ပုံစံဖြင့် ပြန်ပေးစေလိုကြောင်း သတ်မှတ်ခြင်း (ဥပမာ: Bullet Points၊ ဇယား၊ အီးမေးလ်ပုံစံ)။ | **ဖော်မတ်:** "အဖြေကို bullet points သို့မဟုတ် ဇယားပုံစံဖြင့်သာ ပြန်ဖြေပါ"။ | အဖြေကို လွယ်ကူစွာ အသုံးပြုနိုင်ရန် Format သည် အရေးကြီးသည်။ Format မသတ်မှတ်ပါက AI သည် စာပိုဒ်ရှည်ကြီးများဖြင့် ပြန်ဖြေနိုင်ခြေရှိသည်။ |

#### **L 2.2. Zero-shot & Few-shot Prompting: နမူနာပေး၍ တောင်းဆိုနည်း**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Zero-shot Prompting:** AI ကို နောက်ခံအခြေအနေ သို့မဟုတ် နမူနာများ မပေးဘဲ တိုက်ရိုက်မေးမြန်းခြင်း။ ၎င်းသည် ရိုးရှင်းသော မေးခွန်းများ သို့မဟုတ် အချက်အလက်များအတွက် သင့်လျော်သည်။ | **Prompt:** "နေပြည်တော်မှာ အကြီးဆုံး ဘုရားက ဘာလဲ။" (AI သည် ၎င်း၏ လေ့ကျင့်ရေးဒေတာကို အခြေခံပြီး တိုက်ရိုက်ဖြေသည်။) | ရိုးရှင်းသော အချက်အလက် ရှာဖွေခြင်း သို့မဟုတ် အကျဉ်းချုပ်ခြင်းအတွက် Zero-shot ကို အသုံးပြုနိုင်သည်။ |
| **Few-shot Prompting:** AI အား သင်လိုချင်သည့် အလုပ်တာဝန်နှင့် ၎င်း၏ ပြီးပြည့်စုံသော နမူနာ (Example/Answer Pair) တစ်ခု သို့မဟုတ် တစ်ခုထက်ပို၍ ပေးအပ်ပြီးနောက်မှ နောက်ထပ် တူညီသော မေးခွန်းကို မေးမြန်းခြင်း။ | **Prompt:** "အောက်ပါ ဖောက်သည် စကားလုံးကို ရွှင်လန်းသော အသံဖြင့် ပြန်ဖြေပါ- (၁) ဝယ်ယူသူ: 'ပစ္စည်းပို့တာ ဘယ်နေ့ရောက်မလဲ?' (အဖြေ) 'မင်္ဂလာပါ၊ အခုပဲ စစ်ဆေးလိုက်ပါမယ်။ နောက်ဆုံးပို့ခဲ့တဲ့ စာအရ [ရက်စွဲ] မှာ ရောက်ပါလိမ့်မယ်နော်!' (၂) ဝယ်ယူသူ: 'ဒီထုတ်ကုန်ကို ဘယ်လိုသုံးရမလဲ။' (အဖြေ) [AI ကို ဖြေခိုင်းသည်]" | Few-shot သည် AI ကို 'သင်ယူစေခြင်း' ဖြစ်သည်။ အဖြေပုံစံ၊ အသံ သို့မဟုတ် စတိုင် (Tone/Style) ကို တိတိကျကျ လိုအပ်သည့် အခါတွင် အကောင်းဆုံး အသုံးဝင်သည်။ |
| **Few-shot ၏ အကျိုးကျေးဇူးများ:** သေးငယ်သော အသေးစိတ်အချက်များ လိုအပ်သည့် အလုပ်များ (ဥပမာ- စာရင်းအင်းပုံစံ၊ ထုတ်ကုန်ဖော်ပြချက်များ) တွင် တိကျမှုကို သိသိသာသာ တိုးမြှင့်ပေးနိုင်သည်။ | နိုင်ငံခြားဘာသာစကားတစ်ခုကို ဘာသာပြန်ရာတွင်၊ ပုံမှန်ဘာသာပြန်ဆိုချက်အစား ဒေသသုံး စကားလုံးများဖြင့် ဘာသာပြန်စေလိုပါက နမူနာပေး၍ သင်ပေးနိုင်သည်။ | Few-shot နည်းလမ်းသည် Prompt Engineering ၏ အရေးအကြီးဆုံးသော နည်းစနစ်များထဲမှ တစ်ခုဖြစ်ပြီး တိကျမှုအတွက် မဖြစ်မနေ လေ့ကျင့်သင့်သည်။ |

#### **L 2.3. Prompt Refinement (တောင်းဆိုချက် ပိုမိုကောင်းမွန်အောင် ပြုလုပ်ခြင်း)**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Iterative Prompting (ထပ်ခါတလဲလဲ ညှိနှိုင်းတောင်းဆိုခြင်း):** ပထမ Prompt တစ်ခုတည်းဖြင့် လိုချင်သောရလဒ် မရရှိသောအခါတွင် AI ကို Follow-up Instructions (နောက်ဆက်တွဲ ညွှန်ကြားချက်) များ ပေးခြင်းဖြင့် အဖြေကို တဖြည်းဖြည်း တိုးတက်ကောင်းမွန်အောင် လုပ်ဆောင်ခြင်း။ | သင်သည် စားသောက်ဆိုင် မီနူးတစ်ခုကို ရေးခိုင်းသည်။ AI သည် ရှည်လျားလွန်းသော ဖော်ပြချက်များကို ပေးသည်။ ထိုအခါ သင်သည် "ကျေးဇူးပြု၍ ဖော်ပြချက်တိုင်းကို စကားလုံး ၁၀ လုံးထက် မပိုစေဘဲ ပြန်ပြင်ပေးပါ" ဟု ညွှန်ကြားချက် ထပ်ပေးနိုင်သည်။ | AI နှင့် စကားပြောဆိုခြင်းသည် လူသားများနှင့် ပြောဆိုခြင်းကဲ့သို့ပင်ဖြစ်သည်။ ပိုမိုကောင်းမွန်သော ညွှန်ကြားချက်များဖြင့် သင်သည် ၎င်း၏ အလုပ်ကို တိုးတက်အောင် လုပ်နိုင်သည်။ |
| **Constraints (ကန့်သတ်ချက်များ) ပေးခြင်း:** အဖြေ၏ အရှည်၊ အသံ (Tone)၊ ပရိသတ် (Audience) သို့မဟုတ် ဖော်မတ်တို့ကို ကန့်သတ်ချက်များ သတ်မှတ်ပေးခြင်း။ | **ကန့်သတ်ချက် ဥပမာ:** "အသက် ၅ နှစ်ကလေး နားလည်အောင် ရှင်းပြပါ။" သို့မဟုတ် "စကားလုံး ၂၀၀ ထက်မပိုရပါ။" | ကန့်သတ်ချက်များကို ထည့်သွင်းခြင်းသည် AI ၏ အဖြေကို ပိုမိုတိကျစေပြီး မိမိလိုချင်သော ဘောင်အတွင်းသို့ ရောက်စေသည်။ |
| **Editing the Conversation:** Chat interface များ (ဥပမာ- ChatGPT) တွင် မူရင်း Prompt ကို တိုက်ရိုက် ပြင်ဆင်ခြင်းဖြင့် အဖြေအသစ်ကို ချက်ချင်း ထုတ်ယူနိုင်ခြင်း။ ၎င်းသည် အချိန်ကုန်သက်သာစေသည်။ | သင်သည် မူရင်း Prompt တွင် "Formal Tone" ဟု ရေးထားမိပြီး၊ နောက်မှ "Informal Tone" ဖြစ်ချင်ပါက မူရင်း Prompt ကို တိုက်ရိုက် ပြင်ပြီး အဖြေအသစ် ထုတ်နိုင်သည်။ | Prompt Refinement သည် Prompt Engineering ၏ လက်တွေ့ အသုံးချမှု အများဆုံးဖြစ်သည်။ ၎င်းသည် သင်၏ AI ညှိနှိုင်းမှု စွမ်းရည်ကို မြှင့်တင်ပေးသည်။ |

### Module 3: အဆင့်မြင့် Prompting နည်းလမ်းများနှင့် အသုံးချမှု

#### **L 3.1. Role-Based Prompting: AI အား ကျွမ်းကျင်သူအဖြစ် သရုပ်ဆောင်စေခြင်း**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Role Assignment:** AI အား စတင်မေးမြန်းခြင်း မပြုမီ သီးခြားကျွမ်းကျင်သူ တစ်ဦး၏ အခန်းကဏ္ဍ (Persona) ကို သတ်မှတ်ပေးခြင်း။ ၎င်းသည် AI ၏ အသိပညာကို သတ်မှတ်ထားသော နယ်ပယ်တစ်ခုတွင် အာရုံစိုက်စေသည်။ | **Role Prompt ဥပမာ:** "သင်သည် ယခုအခါ အတွေ့အကြုံ ၂၀ နှစ်ရှိ Marketing Strategy Consultant တစ်ဦးဖြစ်သည်။ မြန်မာ့စျေးကွက်နှင့် ပတ်သက်ပြီး နက်နဲသော နားလည်မှုရှိသည်။" | Role-Based Prompting သည် အဖြေ၏ အရည်အသွေးနှင့် ပရော်ဖက်ရှင်နယ်ဆန်မှုကို သိသိသာသာ တိုးတက်စေသည်။ |
| **Benefit of Roles:** AI သည် သာမန်အားဖြင့် စွယ်စုံရ (Generalist) အဖြစ် ဖြေဆိုသော်လည်း Role သတ်မှတ်လိုက်သောအခါ ကျွမ်းကျင်သူတစ်ဦးကဲ့သို့ စဉ်းစားတွေးခေါ်ကာ သက်ဆိုင်ရာ ဝေါဟာရများနှင့် နည်းဗျူဟာများကို အသုံးပြုလာသည်။ | သာမန် AI သည် "ဆီမီးကပ်" ပေးပါ။ ဟု ဖြေသော်လည်း၊ "SEO Specialist" Role ကို ပေးလိုက်ပါက AI သည် "SEO အတွက် Keyword Research ကို အသေးစိတ် လုပ်ဆောင်ပါမည်" ဟု ပိုမိုတိကျသော အဖြေ ပေးလာမည်။ | ပရော်ဖက်ရှင်နယ် အဖြေများ လိုအပ်ပါက Role ကို သတ်မှတ်ပါ။ Context ကို ထည့်သွင်းပေးခြင်းဖြင့် Role ကို ပိုမိုအားကောင်းစေနိုင်သည်။ |
| **Effective Role Types:** စီးပွားရေးလုပ်ငန်းများအတွက် အသုံးဝင်သော Roles များမှာ Marketing Manager, Financial Analyst, Creative Copywriter, Software Developer နှင့် Customer Service Representative တို့ဖြစ်သည်။ | Role-Based Prompting သည် သင်၏ လုပ်ငန်းခွင်ပြဿနာများကို ကျွမ်းကျင်သူတစ်ဦးနှင့် တိုင်ပင်ဆွေးနွေးသကဲ့သို့ ခံစားရစေသည်။ | AI သည် အမှန်တကယ် ကျွမ်းကျင်သူမဟုတ်သော်လည်း၊ ထို Role မှ သင်ယူခဲ့သော ဒေတာများကို အသုံးပြု၍ အကောင်းဆုံး အဖြေကို ထုတ်ပေးသည်။ |

#### **L 3.2. Chain-of-Thought (CoT) Prompting: အဆင့်ဆင့် တွေးခေါ်စေခြင်း**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Chain-of-Thought (CoT) Prompting:** AI အား အဖြေပေးရန် တိုက်ရိုက်မတောင်းဆိုမီ ကြားခံအဆင့်များ (Intermediate Steps) ဖြင့် ၎င်း၏ တွေးခေါ်ပုံ သို့မဟုတ် ပြဿနာဖြေရှင်းပုံကို ဖော်ပြရန် တောင်းဆိုခြင်း။ | သင်္ချာပြဿနာတစ်ခုကို တွက်ချက်ခိုင်းခြင်းထက်၊ "အဖြေကို မပေးခင် တွက်ချက်ပုံ အဆင့်ဆင့်ကို အရင်ရေးပြပါ" ဟု ပြောခြင်းသည် CoT ဖြစ်သည်။ | CoT သည် AI ၏ အမှားများကို လျော့ချပေးပြီး ရှုပ်ထွေးသော ဆုံးဖြတ်ချက်ချမှုများ (Complex Reasoning) အတွက် မရှိမဖြစ် လိုအပ်သည်။ |
| **How CoT Improves Accuracy:** AI သည် ရှုပ်ထွေးသော မေးခွန်းများကို ဖြေဆိုရာတွင် အဆင့်တိုင်းကို စစ်ဆေးခြင်းဖြင့် ပထမအဆင့်မှ အမှားသည် နောက်အဆင့်သို့ ကူးစက်ခြင်းကို ကာကွယ်ပေးသည်။ | လမ်းညွှန်ပြသပေးသည့် မြေပုံနှင့် တူသည်။ လမ်းဆုံးကို တိုက်ရိုက်ပြမည့်အစား၊ တစ်ကွေ့ချင်းစီကို ဖြတ်သန်းပုံ (Step 1, Step 2, Step 3) ကို ပြသခြင်းဖြင့် လမ်းပျောက်ခြင်းကို လျှော့ချသည်။ | CoT ကို အဓိကအားဖြင့် ခွဲခြမ်းစိတ်ဖြာခြင်း၊ တွက်ချက်ခြင်း၊ အစီအစဉ်ဆွဲခြင်း (Planning) နှင့် ယုတ္တိဗေဒဆိုင်ရာ မေးခွန်းများအတွက် အသုံးပြုသင့်သည်။ |
| **Techniques for CoT:** Prompt တွင် "Let's think step by step," သို့မဟုတ် "Before giving the final answer, outline your logical process" ကဲ့သို့သော စကားစုများ ထည့်သွင်းခြင်း။ | **Prompt ဥပမာ (စီးပွားရေး):** "ဤဒေတာဇယားကို ခွဲခြမ်းစိတ်ဖြာရန် အဆင့် ၃ ဆင့်ကို အရင်ဆုံး စာရင်းပြုစုပါ၊ ထို့နောက်မှ အဆင့်တိုင်းကို လိုက်နာ၍ ရလဒ်ကို ပေးပါ။" | CoT သည် AI ကို ပိုမို စေ့စပ်သေချာပြီး ယုတ္တိကျသော ဖြေဆိုမှု ပုံစံသို့ ပြောင်းလဲပေးသည်။ |

#### **L 3.3. AI ကို အကြောင်းအရာ ဖန်တီးရာတွင် အသုံးပြုခြင်း (Content Generation)**

| Key Concepts | Example or Analogy | Key Takeaways |
| :--- | :--- | :--- |
| **Content Structuring:** AI ကို အချက်အလက်အပြည့်အစုံ ရေးသားခိုင်းခြင်းထက်၊ အကြောင်းအရာ၏ ဖွဲ့စည်းပုံ (Outline) သို့မဟုတ် အပိုင်းပိုင်းခွဲခြင်းအတွက် အသုံးပြုခြင်း။ | အကြောင်းအရာ တစ်ခုလုံးကို တစ်ပြိုင်တည်း ရေးခိုင်းမည့်အစား၊ ပထမဦးစွာ "Facebook Post ၅ ခုအတွက် ခေါင်းစဉ် (Title) များကိုသာ ထုတ်ပေးပါ" ဟု တောင်းဆိုနိုင်သည်။ | အဆင့်လိုက် ဖန်တီးခြင်း (Structure -> Draft -> Edit) သည် AI ၏ အမှားကို လျှော့ချပြီး အချိန်ကုန်သက်သာစေသည်။ |
| **Drafting & Tone Adjustment:** AI အား ပထမမူကြမ်း (First Draft) ကို ရေးဆွဲခိုင်းပြီး၊ ထိုမူကြမ်းကို မိမိလိုချင်သော အသံ (Tone) နှင့် ပရိသတ် (Audience) နှင့် ကိုက်ညီအောင် ပြန်လည်ညှိနှိုင်းခိုင်းခြင်း။ | သင်သည် ထုတ်ကုန်အသစ်တစ်ခုအကြောင်း အချက်အလက်များ ထည့်သွင်းပြီး AI ကို "ဖောက်သည်ကို စွဲဆောင်နိုင်သော ဖော်မတ်ဖြင့် အီးမေးလ် မူကြမ်းကို ရေးပါ" ဟု ခိုင်းစေနိုင်သည်။ | AI သည် ကနဦး မူကြမ်းများ ရေးသားရာတွင် ထူးချွန်သည်။ လူသား၏ အခန်းကဏ္ဍမှာ Editing (တည်းဖြတ်ခြင်း) နှင့် Verification (အတည်ပြုခြင်း) ဖြစ်သည်။ |
| **Human-in-the-Loop Editing:** AI မှ ထုတ်ပေးသော Content အားလုံးကို မထုတ်ပြန်မီ စစ်ဆေးခြင်း၊ အမှားပြင်ဆင်ခြင်းနှင့် ဒေသခံ ယဉ်ကျေးမှု သို့မဟုတ် အခြေအနေနှင့် ကိုက်ညီမှု ရှိမရှိကို သေချာအောင် ပြန်လည် ထည့်သွင်းခြင်း။ | AI သည် မြန်မာ့ယဉ်ကျေးမှုနှင့် မရင်းနှီးသော အသုံးအနှုန်းများ သို့မဟုတ် နိုင်ငံခြား အယူအဆများကို ထည့်သွင်းမိနိုင်သည်။ ထို့ကြောင့် လူသားများက ပြန်လည် စိစစ်တည်းဖြတ်ပေးရမည်။ | AI Content ကို Copy & Paste မလုပ်ဘဲ၊ တည်းဖြတ်ခြင်း (Editing) နှင့် ကိုယ်ပိုင်အမြင် (Personal Touch) ထည့်သွင်းခြင်းကို မမေ့ပါနှင့်။ |

1.  သင်ကိုယ်တိုင် သို့မဟုတ် စိတ်ကူးဖြင့် ရွေးချယ်ထားသော စီးပွားရေးလုပ်ငန်း (ဥပမာ- လက်မှုပစ္စည်းဆိုင်) နှင့် ထုတ်ကုန် ၅ မျိုးကို သတ်မှတ်ပါ။
2.  သင်အသုံးပြုမည့် AI tool (ChatGPT, Gemini, သို့မဟုတ် Claude) ကို ရွေးချယ်ပါ။
3.  သင်လိုချင်သည့် ထုတ်ကုန်ဖော်ပြချက်၏ စတိုင် (Tone and Style) နှင့် အရှည် (Length) ကို သတ်မှတ်ပါ။ (ဥပမာ- 'ဖော်ရွေသော လေသံ၊ စကားလုံး ၂၅ လုံးထက် မပိုရ')
4.  Few-shot နည်းစနစ်ကို အသုံးပြုရန်၊ သင်ကိုယ်တိုင် စံနမူနာ (Example) ၏ Input နှင့် Output တွဲများ (Example/Answer Pair) ကို အနည်းဆုံး နှစ်စုံ ရေးသားပါ။
    *   *Input 1 (Original Product Idea):* "[ထုတ်ကုန်အမည်]"
    *   *Output 1 (Desired Style/Length):* "[လိုချင်သော ပုံစံဖြင့် ရေးသားထားသည့် ဖော်ပြချက်]"
5.  Prompt အပြီးအပြည့်အစုံကို (Instruction, Context, Example, Format) အခြေခံ ၄ ချက်ဖြင့် ရေးဆွဲပါ။ (Example အဖြစ် step 4 မှ နမူနာများကို ထည့်ပါ။)
6.  AI ကို အသုံးပြု၍ ကျန်ရှိသော ထုတ်ကုန် ၄ ခုအတွက် ဖော်ပြချက်များကို အသစ်ထုတ်ပေးရန် တောင်းဆိုပါ။
7.  ရလာသော အဖြေများကို စစ်ဆေးပါ။ မူလ စတိုင်နှင့် ကိုက်ညီမှုမရှိပါက Prompt Refinement (L 2.3) ကို အသုံးပြု၍ အဖြေများ ပိုမိုကောင်းမွန်လာစေရန် နောက်ဆက်တွဲ ညွှန်ကြားချက်များ (ဥပမာ- "ပိုမိုဆွဲဆောင်မှုရှိသော စကားလုံးများဖြင့် ပြင်ဆင်ပါ") ပေးပါ။
8.  အကောင်းဆုံးရလဒ် ၅ ခုကို စုစည်းပါ။

**Starter hints:**

*   Prompt ရေးရာတွင် "Context" အပိုင်း၌ သင့်လုပ်ငန်း၏ ပရိသတ် (Audience) (ဥပမာ- လူငယ်များ သို့မဟုတ် မိဘများ) ကို သတ်မှတ်ပေးပါ။
*   Few-shot နမူနာများကို စတင်ရန်၊ ခေါင်းစဉ် (Title) နှင့် စာကြောင်း ၂ ကြောင်း ပါဝင်သော စံတစ်ခုကို အသုံးပြုပါ။

**Expected Deliverables:**

1.  သင်အသုံးပြုခဲ့သော Few-shot Prompt အပြည့်အစုံ (အခြေခံ ၄ ချက်ပါဝင်သော) ကို Screenshot သို့မဟုတ် Text File ဖြင့် တင်ပြပါ။
2.  ထုတ်ကုန် ၅ ခုအတွက် AI မှ ထုတ်ပေးပြီး Refinement ပြုလုပ်ထားသော နောက်ဆုံး Product Descriptions များကို ဖော်ပြပါ။
3.  Assignment ၏ နောက်ဆက်တွဲအနေဖြင့်၊ AI ကို မူလ Prompt ဖြင့် မေးရာတွင် မည်သည့်ပြဿနာများ ကြုံတွေ့ခဲ့ပြီး Prompt Refinement ဖြင့် မည်သို့ ဖြေရှင်းခဲ့ကြောင်းကို စာကြောင်း ၄ ကြောင်းဖြင့် ရှင်းပြပါ။

**Common pitfalls and tips:**

1.  **Pitfall:** Few-shot နမူနာများနှင့် နောက်ဆုံးလိုချင်သော ရလဒ် ပုံစံချင်းမတူခြင်း။ **Tip:** နမူနာ Output များသည် လိုချင်သော Tone, Length, Format တို့ကို တိကျစွာ ကိုယ်စားပြုရမည်။
2.  **Pitfall:** Prompt Refinement ကို မပြုလုပ်ဘဲ ပထမဆုံး အဖြေဖြင့် ကျေနပ်ခြင်း။ **Tip:** Prompt Refinement (L 2.3) သည် သင်ယူမှု၏ အဓိကအစိတ်အပိုင်းဖြစ်သဖြင့်၊ ပထမအဖြေသည် မပြည့်စုံသည့်တိုင် နောက်ဆက်တွဲ ညွှန်ကြားချက်များ ပေးခြင်းကို လေ့ကျင့်ပါ။

### Assignment 2: လုပ်ငန်းခွင်သုံး AI Project (Module 4 အပြီး)

| Detail | Description |
| :--- | :--- |
| **Assignment Title** | Role-Based, CoT နှင့် Automation ကို ပေါင်းစပ်၍ စီးပွားရေးလုပ်ငန်း လည်ပတ်မှု ပုံစံ (Workflow) ကို မြှင့်တင်ခြင်း |
| **Description** | သင်၏ စိတ်ကူးယဉ် အွန်လိုင်းစီးပွားရေးလုပ်ငန်းတစ်ခုအတွက် အပတ်စဉ် စီမံခန့်ခွဲမှု အလုပ် (Management Task) တစ်ခုကို AI ဖြင့် အလိုအလျောက် (Automate) ပြုလုပ်ရန် Prompt စီးရီးတစ်ခုကို တည်ဆောက်ပါ။ ဤ Project တွင် Role-Based Prompting (L 3.1)၊ Chain-of-Thought (L 3.2) နှင့် Data Summarization (L 4.1) တို့ကို ပေါင်းစပ်အသုံးပြုရမည်။ |
| **Estimated Effort** | 5 နာရီ |
| **Learning Goals Addressed** | Chain-of-thought နှင့် role-based prompting ဖြင့် ရလဒ်ကောင်းများ ရယူနိုင်ရန်။ AI Tools များကို Business Workflow Automation တွင် လက်တွေ့ အသုံးချနိုင်ရန်။ AI အသုံးပြုရာတွင် ကျင့်ဝတ်နှင့် ကန့်သတ်ချက်များကို သိရှိ နားလည်နိုင်ရန်။ |

**Scenario:** သင်၏အွန်လိုင်းစီးပွားရေးလုပ်ငန်းသည် လူမှုကွန်ရက် Comment များမှ ဖောက်သည်တုံ့ပြန်ချက် ၅ ခုကို စုဆောင်းထားသည်။ သင်သည် ထိုတုံ့ပြန်ချက်များကို အသေးစိတ် ခွဲခြမ်းစိတ်ဖြာပြီး၊ လာမည့်အပတ်အတွက် မားကတ်တင်း ဗျူဟာကို ဆုံးဖြတ်လိုသည်။

**Step-by-step instructions:**

1.  **Preparation:** အောက်ပါအတိုင်း စိတ်ကူးဖြင့် ရေးသားထားသော ဖောက်သည် Comment ၅ ခုကို ဖန်တီးပါ။ (အနည်းဆုံး Comment ၁ ခုသည် အနုတ်လက္ခဏာ (Negative) ဖြစ်ပြီး၊ ၁ ခုသည် အပေါင်းလက္ခဏာ (Positive) ဖြစ်ရမည်။)
2.  **Role Assignment:** AI အား "အတွေ့အကြုံရှိ Customer Insight Analyst တစ်ဦးအဖြစ် သရုပ်ဆောင်ပါ" ဟူ၍ Role Prompt (L 3.1) ကို ရေးသားပါ။
3.  **CoT Prompt Design (Phase 1 - Analysis):** Comment များကို တိကျစွာ ခွဲခြမ်းစိတ်ဖြာရန် CoT နည်းစနစ်ကို အသုံးပြု၍ Prompt ရေးပါ။
    *   *Prompt Instruction:* "အောက်ပါ Comment ၅ ခုကို ခွဲခြမ်းစိတ်ဖြာပြီး၊ အဖြေကို မပေးမီ 'Step-by-Step' တွေးပါ။"
    *   *CoT Step:* (၁) Comment တိုင်း၏ Tone (အသံ- Positive, Negative, Neutral) သတ်မှတ်ပါ။ (၂) ဖောက်သည်များ၏ အဓိက လိုအပ်ချက် ၃ ခုကို စာရင်းပြုစုပါ။ (၃) ဤတုံ့ပြန်ချက်များ၏ အကြီးမားဆုံး အခွင့်အလမ်း (Opportunity) ကို သတ်မှတ်ပါ။
4.  **Automation & Summarization (Phase 2 - Strategy):** Phase 1 မှ ရရှိလာသော ခွဲခြမ်းစိတ်ဖြာချက်ကို အခြေခံ၍ ဒုတိယ Prompt ကို ရေးသားပါ။
    *   *Prompt Instruction:* "Phase 1 မှ အဖြေများကို အခြေခံ၍ လာမည့်အပတ်အတွက် အရေးကြီးဆုံး Marketing Strategy ၃ ချက်ကို စာရင်းပြုစုပါ။"
    *   *Format:* အဖြေကို ဇယားပုံစံဖြင့် ဖော်ပြပြီး Strategy တိုင်းအတွက် အဘယ်ကြောင့် အရေးကြီးသည်ကို ရှင်းပြပါ။
5.  **Ethical Review (Phase 3):** နောက်ဆုံး Prompt တစ်ခုဖြင့် AI ကို ကျင့်ဝတ်ဆိုင်ရာ စိစစ်ခိုင်းပါ။
    *   *Prompt Instruction:* "ဤခွဲခြမ်းစိတ်ဖြာမှုတွင် Hallucination ဖြစ်နိုင်ခြေရှိသော အချက်အလက်များ ပါဝင်ပါသလား။ Data Privacy ကို ထိခိုက်မှုရှိပါသလား။ (L 4.3 ကို အခြေခံ၍) သင်၏ အမြင်ကို ရှင်းပြပါ။"
6.  **Submission:** Prompt စီးရီး ၃ ခုလုံးနှင့် ၎င်းတို့၏ AI Output များကို စုစည်းပါ။

**Starter hints:**

*   Phase 1 (Analysis) သည် CoT အတွက် အရေးကြီးသည်။ AI ကို ရှည်လျားသော စာသားများ အစား ဇယားများဖြင့် အဖြေပြန်ပေးရန် Format ကို သတ်မှတ်ပါ။
*   ဖောက်သည် Comments များကို Prompt ထဲသို့ ကူးထည့်သောအခါ၊ ၎င်းတို့သည် လျှို့ဝှက်အချက်အလက်မဟုတ်ကြောင်း သေချာပါစေ။

**Expected Deliverables:**

1.  ဖန်တီးထားသော ဖောက်သည် Comments ၅ ခု။
2.  Prompt 1 (Role + CoT Analysis)၊ Prompt 2 (Automation/Strategy)၊ Prompt 3 (Ethical Review) တို့၏ Text သို့မဟုတ် Screenshot များ။
3.  Prompt 3 ၏ AI Output အပေါ်တွင် သင်၏ ကိုယ်ပိုင် အဆုံးသတ် မှတ်ချက် (Conclusion) ကို စာပိုဒ်တစ်ခုဖြင့် ရေးသားပါ။ (AI ၏ အဖြေကို စိစစ်တည်းဖြတ်ရန် Human-in-the-Loop အဆင့်ကို ဖော်ပြရမည်။)

**Common pitfalls and tips:**

1.  **Pitfall:** Role-Based နှင့် CoT ကို ပေါင်းစပ်မသုံးခြင်း။ **Tip:** Prompt 1 သည် Role Assignment ဖြင့် စတင်ပြီး "Let's think step by step" ဖြင့် ဆက်လက် ညွှန်ကြားရမည်။
2.  **Pitfall:** Prompt 2 တွင် Phase 1 ၏ ခွဲခြမ်းစိတ်ဖြာမှုရလဒ်များကို ပြန်လည် ထည့်သွင်းရန် မေ့လျော့ခြင်း။ **Tip:** LLM ၏ Context Window ကန့်သတ်ချက်ကြောင့်၊ ပထမ Phase မှ အရေးကြီးသော Key Findings များကို ဒုတိယ Prompt တွင် ထပ်မံ Context အဖြစ် ထည့်သွင်းသင့်သည်။
3.  **Pitfall:** Ethical Review ကို အပေါ်ယံသာ မေးမြန်းခြင်း။ **Tip:** Hallucination နှင့် Data Privacy တို့သည် အဓိက ခေါင်းစဉ်များဖြစ်သောကြောင့်၊ AI သည် ထိုအန္တရာယ်များကို မည်သို့ရှောင်ရှားနိုင်သည်ကို အသေးစိတ် မေးမြန်းပါ။

Effective Use of Advanced Techniques (Role & CoT)** | 30% | Role-Based Assignment ကို ထိရောက်စွာ အသုံးပြုထားပြီး CoT (အဆင့်ဆင့်တွေးခေါ်မှု) ကို ရှုပ်ထွေးသော ခွဲခြမ်းစိတ်ဖြာမှုများအတွက် ကျွမ်းကျင်စွာ ပေါင်းစပ်အသုံးပြုထားသည်။ | Role Assignment ပါဝင်ပြီး CoT ကို အသုံးပြုထားသော်လည်း အဆင့်ဆင့် ခွဲခြမ်းစိတ်ဖြာမှု (Step-by-Step Reasoning) မှာ အနည်းငယ် အားနည်းနေသည်။ | Role ကို အသုံးပြုထားသော်လည်း CoT မှာ ရိုးရှင်းသော မေးခွန်းများအတွက်သာ ဖြစ်နေခြင်း သို့မဟုတ် Role ၏ သက်ရောက်မှု အားနည်းခြင်း။ | Role-Based သို့မဟုတ် CoT နည်းလမ်းများကို လုံးဝ အသုံးမပြုခြင်း သို့မဟုတ် မှားယွင်းစွာ အဓိပ္ပာယ်ဖွင့်ဆိုခြင်း။ |
| **3. Output Quality and Relevance (Automation)** | 25% | AI မှ ထွက်ပေါ်လာသော Strategy နှင့် Summarization များသည် Scenario နှင့် တိကျစွာ ကိုက်ညီပြီး လုပ်ငန်းခွင်၌ လက်တွေ့ကျသော အသုံးချနိုင်မှုရှိသည်။ | AI Output များသည် လိုအပ်ချက်နှင့် ကိုက်ညီသော်လည်း အသေးစိတ်အချက်အလက် သို့မဟုတ် ပရော်ဖက်ရှင်နယ်ဆန်မှု (Professionalism) အနည်းငယ် လိုအပ်နေသည်။ | AI Output များသည် ရှင်းလင်းသော အဖြေပေးသော်လည်း Prompt ၏ ညွှန်ကြားချက်များကို လုံးဝအတိုင်း မလိုက်နာခြင်း သို့မဟုတ် အချို့အချက်များ မပါဝင်ခြင်း။ | AI မှ ထုတ်ပေးသော အဖြေများတွင် Hallucination ဖြစ်ခြင်း သို့မဟုတ် Prompt ၏ ရည်ရွယ်ချက်နှင့် လုံးဝ မကိုက်ညီခြင်း။ |

## IV. FEEDBACK COMMENT TEMPLATES (အကြံပြုချက် နမူနာပုံစံများ)

### Performance Level: Excellent (ထူးချွန်)

1.  **[Specific Action/Criterion]:** သင်၏ Prompt စီးရီးများသည် အလွန်ရှင်းလင်းပြတ်သားပြီး Few-shot မှ Role-Based သို့ ကူးပြောင်းအသုံးပြုခြင်းသည် အဖြေ၏ အရည်အသွေးကို သိသိသာသာ မြှင့်တင်ပေးပါသည်။ Advanced Prompting နည်းစနစ်များကို ကျွမ်းကျင်စွာ ပေါင်းစပ်အသုံးပြုနိုင်သည်ကို တွေ့ရသည်။
2.  **[Strategy/Technique]:** Chain-of-Thought (CoT) ကို အသုံးပြု၍ Customer Comment များကို ခွဲခြမ်းစိတ်ဖြာခဲ့သည့် အဆင့်ဆင့် တွေးခေါ်မှု (Step-by-Step Reasoning) မှာ အလွန်ပြည့်စုံသောကြောင့် နောက်ဆုံး Strategy သည် တိကျမှန်ကန်ပါသည်။ ရှုပ်ထွေးသော ပြဿနာများကို ဖြေရှင်းရာတွင် ဤနည်းလမ်းကို ဆက်လက်အသုံးပြုပါ။
3.  **[Ethics/Application]:** Ethical Review အပိုင်းတွင် Hallucination နှင့် Data Privacy အန္တရာယ်များကို ကောင်းစွာ နားလည်ထားပြီး လုပ်ငန်းခွင်၌ တာဝန်ယူမှုရှိစွာ အသုံးပြုနိုင်ရန် သင်၏ ကိုယ်ပိုင်အဆုံးသတ် မှတ်ချက်သည် အလွန်ခိုင်မာပါသည်။

### Performance Level: Good (ကောင်းမွန်)

1.  **[Specific Action/Criterion]:** Role-Based Prompting နှင့် Automation Prompt များသည် အလွန်အားကောင်းပြီး လုပ်ငန်းလိုအပ်ချက်များနှင့် ကိုက်ညီသည်။ AI မှ ထုတ်ပေးသော Content များသည် သက်ဆိုင်ရာ Role မှ လာသည်ဟု ခံစားရစေသည်။
2.  **[Strategy/Technique]:** Prompt Refinement အဆင့်တွင် သင်၏ ညွှန်ကြားချက်များသည် ထိရောက်သော်လည်း၊ Prompt 2 တွင် Phase 1 မှ အဖြေ၏ အဓိကအချက်အလက်များကို Context အဖြစ် ထပ်မံထည့်သွင်းပေးခြင်းဖြင့် AI ၏ မှတ်ဉာဏ်ကန့်သတ်ချက် (Context Window) ကို ကျော်လွှားနိုင်ရန် ပိုမိုကောင်းမွန်စေပါသည်။
3.  **[Ethics/Application]:** Ethical Review Prompt သည် မှန်ကန်သော်လည်း Hallucination ဖြစ်နိုင်ခြေကို ဖော်ပြရာတွင် AI ၏ ဘက်လိုက်မှု (Bias) ကဲ့သို့သော အခြားကျင့်ဝတ်ဆိုင်ရာ ပြဿနာများကိုပါ ထည့်သွင်းစဉ်းစားရန် အကြံပြုလိုပါသည်။

### Performance Level: Satisfactory (ကျေနပ်ဖွယ်)

1.  **[Specific Action/Criterion]:** Assignment ကို အချိန်မီ တင်ပြနိုင်ပြီး Prompt ၏ အခြေခံအစိတ်အပိုင်းများ (Instruction, Context) ကို အသုံးပြုနိုင်သည်ကို တွေ့ရသည်။ အထူးသဖြင့် Data Summarization အပိုင်းသည် အလုပ်ဖြစ်ပါသည်။
2.  **[Strategy/Technique]:** Role Assignment (L 3.1) ကို အသုံးပြုထားသော်လည်း CoT ကို ထိရောက်စွာ အသုံးမပြုနိုင်ပါ။ ရှုပ်ထွေးသော ခွဲခြမ်းစိတ်ဖြာမှုများအတွက် CoT ညွှန်ကြားချက် ("Let's think step by step") ကို ပိုမို ရှင်းလင်းစွာ ထည့်သွင်းပြီး AI ကို အဆင့်ဆင့် တွက်ချက်စေရန် ထပ်မံ လေ့ကျင့်ပါ။
3.  **[Ethics/Application]:** AI Output များတွင် Format (ဇယား သို့မဟုတ် Bullet Points) ကို မသတ်မှတ်ခဲ့သောကြောင့် ရလဒ်သည် စာပိုဒ်ရှည်များအဖြစ် ထွက်လာသည်။ Prompt ရေးသားရာတွင် Format Component ကို မမေ့မလျော့ ထည့်သွင်းပေးရန် လိုအပ်ပါသည်။

### Performance Level: Needs Improvement (တိုးတက်ရန် လိုအပ်)

1.  **[Specific Action/Criterion]:** Assignment ၏ အဓိက လိုအပ်ချက်များဖြစ်သော Advanced Prompting နည်းလမ်းများ (Role-Based, CoT) ကို လုံးဝနီးပါး အသုံးမပြုထားသည်ကို တွေ့ရသည်။
2.  **[Strategy/Technique]:** Prompt များသည် ရိုးရှင်းသော Zero-shot ပုံစံများသာ ဖြစ်နေသည်။ Module 2 (L 2.1) တွင် သင်ကြားခဲ့သော Prompt ၏ ဖွဲ့စည်းပုံ (၄ ချက်) ကို သေချာပြန်လည် လေ့လာပါ။ Context နှင့် Instruction ကို ခွဲခြားပြီး ရှင်းလင်းပြတ်သားစွာ ရေးသားတတ်ရန် လိုအပ်ပါသည်။
3.  **[Ethics/Application]:** Ethical Review အပိုင်း မပါဝင်ပါ။ AI ကို အသုံးပြုရာတွင် Hallucination ကဲ့သို့သော ကန့်သတ်ချက်များကို အမြဲတမ်း သတိပြုရန်နှင့် လျှို့ဝှက်ဒေတာများကို မည်သည့်အခါမျှ မထည့်သွင်းရန် L 4.3 ကို အလေးထား လေ့လာရန် အကြံပြုအပ်ပါသည်။
